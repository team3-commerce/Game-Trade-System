# 비즈니스 로직

## 회원

- 회원 탈퇴가 가능하다.
    - 회원이 탈퇴할 경우 거래장에는 판매중인 상품과 수령 대기 항목이 없어야 한다.
    - 회원이 탈퇴하기 전 상품을 팔았을 경우, 그리고 구매자가 그 상품을 수령 전이라면 구매자는 여전히 수령 할 수 있다.
- 닉네임은 다른 사람과 중복되지 않은 닉네임으로 변경 가능하다.
- 비밀번호는 변경 가능하다.
- 비활성화 상태는 관리자가 정지를 시키는 경우와 장기간 미접속의 경우로 따로 관리한다.
- 장기간 미접속의 경우 로그인을 하면 계정이 활성화 상태로 즉시 변경된다.
- 30일 미접속의 경우 비활성화 상태로 변경된다.
- 회원탈퇴를 하면 1년 후에 모든 데이터가 삭제된다.

## 로그인

### 통합 인증 수단 관리

- **인증 테이블 분리 혹은 확장** : Member 엔티티는 이메일을 고유값으로 가지되, 어떤 경로(Provider)로 가입했는지 정보를 저장한다.
- **다중 연동 허용** : 한 명의 유저가 Google로 가입했더라도, 나중에 마이페이지에서 Kakao나 Github을 연동하여 여러 수단으로 로그인할 수 있게 구현한다.

### 소셜 가입자의 일반 로그인 전환 (비밀번호 설정)

소셜 가입자도 언제든 아이디/비밀번호 방식을 사용할 수 있어야 한다.

- **비밀번호 설정 기능** : 소셜로 처음 가입한 유저는 비밀번호가 없는 상태이다. 이 유저가 일반 로그인을 시도하려면 '비밀번호 설정' 과정을 거치게 한다.
- **일반 로그인 가능** : 비밀번호가 설정되는 순간, 해당 유저는 소셜 로그인과 이메일 + 패스워드 로그인 두 가지를 모두 사용할 수 있는 '통합 계정'이 된다.

### 소셜 연동 해제

- **상태** : 우리 서비스 회원 탈퇴가 아니라, 단지 구글/카카오와의 연결만 끊는 것이다.
- **로직** : Member 테이블에서 해당 소셜의 providerId를 삭제한다.
- **주의** : 만약 다른 로그인 수단(일반 로그인 등)이 없다면 연동 해제를 막거나, 반드시 비밀번호를 먼저 설정하도록 유도해야 계정 분실을 막을 수 있다.

### 소셜 서비스 탈퇴

- **상황** : 유저가 구글이나 카카오 자체를 탈퇴한 경우이다.
- **로직** : 우리 서버는 소셜 서버로부터 실시간 탈퇴 알림을 받기 어렵다. 따라서 유저가 우리 서비스에 다시 소셜 로그인을 시도할 때 인증 실패가 발생하게 되며, 이때 "연동된 소셜 계정을 찾을 수 없습니다. 일반 로그인을 이용해 주세요."와 같은 가이드를 제공한다.
 
### 이메일 기반 통합 로직

- **최초 가입 시** : 이메일 중복 체크를 수행한다. 이미 같은 이메일로 가입된 '일반 회원'이 소셜 로그인을 시도하면, 신규 가입 대신 "기존 계정에 소셜 계정을 연동하시겠습니까?"라고 묻는 프로세스를 구축한다.
- **데이터 일관성** : 어떤 수단으로 들어오든 동일한 Member ID를 반환하여 장바구니, 주문 내역 등이 유지되도록 한다.

## 아이템

- 아이템의 종류는 장비, 소비, 기타가 존재한다.
- 모든 아이템은 교환이 가능하다.
- 아이템의 가격은 정해져 있지 않다.

## 인벤토리

- 인벤토리의 크기는 무한하다.
    - 모두 소모한 아이템의 수량은 0으로 표기된다.
    - 회원가입 시 기본적인 아이템을 준다.

## 거래소

- 거래소 아이템의 가격은 유저가 정한다.
- 유저는 거래소 아이템의 가격을 아이템 별로 정할 수도 있고, 총 가격으로 정할 수도 있다.
- 유저는 거래소에 등록된 상품의 일부만 살 수 없다.(= 부분 구매 불가)
- 상품 조회 (판매자 / 구매자) → 판매 종료일이 안넘는 상품만 조회
    - 인기 검색어 조회 가능 (검색 순위가 높은 5개까지 조회, 하루 단위로 인기 검색 순위 초기화)
    - 특정 단어로 시작하는 인기 검색어 조회 가능
- 상품 등록 (판매자)
    - 상품 등록 시 한 번에 하나의 상품(품목)만 가능하다.(동일품목 복수의 개수인 경우 수량 입력)
    - 내 등록 상품 조회 가능
    - 내 상품 취소 가능
        - 판매된 상품은 취소하지 못한다. (status가 SELLING인 경우에만 활성화)
        - 상품 판매 취소 시 해당 상품 수령해야 한다. 단, 취소된 상품을 제한 시간동안 수령하지 않을 시 소멸
    - 판매자는 판매 만료 시간을 12시간 단위로 지정할 수 있다. (마켓에 등록할때 12시간, 24시간, 48시간 중에서 선택)
        - 우리는 00시, 06 12 18시에 판매 완료를 확인한다.
            - 새벽 3시에 올린 사람의 경우 오전12시에 만료를 할경우 시장에 12시간 이하의 기간 동안 있었으므로 오후 18시에 만료한다 (실질적으로 시장에 있는 시간은 15시간)
- 상품 구매 (구매자)
    - 상품 세부 사항 확인(개당 금액, 총 금액, 상품 수량, 상품 타입(장비, 소비, 기타))
    - 구매 내역 조회
    - 상품 등록 시간 조회
    - 상품 만료 시간 조회
- 상품 등록 만료 (판매자)
    - 상품 등록 후 제한시간 마감 시 반환한다.  단, 제한 시간동안 반환된 상품을 수령하지 않을 시 소멸
- 거래 후
    - 상품 거래 대금(돈) 수령 가능하다. 단, 거래 대금을 제한 시간동안 수령하지 않을 시 소멸
    - 상품 거래 물품(상품) 수령 가능하다.  단, 거래 상품을 제한 시간동안 수령하지 않을 시 소멸
    

## 쿠폰

- 쿠폰 발행
    - 관리자가 쿠폰 정책을 생성
- 쿠폰 발급
    - 회원가입 시 10000머니를 얻을 수 있는 쿠폰 지급
    - 선착순으로 게임 머니를 얻을 수 있는 선착순 쿠폰 발급
- 선착순 쿠폰 발급 신청
    - 사용자가 쿠폰을 발급하기 위해 신청한다
    - 발급 성공 시 내 쿠폰 리스트에 추가된다
- 쿠폰 정책 조회
    - 쿠폰의 정보 조회 가능
        - 총 수량/남은 수량 조회 가능
    - issue_type 별로 조회 가능
- 내 쿠폰 전체 조회
    - 미사용, 사용, 만료 상태의 쿠폰 모두 조회 가능
    - 쿠폰 상태별 조회 가능
- 내 쿠폰 단건 조회
    - 쿠폰 상태 상관없이 조회 가능
- 내 쿠폰 사용
    - 사용 시 쿠폰 상태를 USED 로 변경 및 쿠폰 사용 기록 추가
    - 내 지갑에 돈 추가 및 지갑 변경 기록 추가
- 내 쿠폰 사용 내역 조회
    - 상태가 사용, 만료인 쿠폰만 조회 가능
- 내 쿠폰 만료
    - 쿠폰 만료일을 확인한 후, 만료 시 상태 EXPIRED 로 변경 및 쿠폰 변경 기록 추가

## 채팅

- 채팅방 생성
    - 채팅방은 구매자가 상품 탭에서 "판매자와 채팅하기"를 통해서만 생성할 수 있다.
    - 채팅방 생성 시 채팅을 시작한 사람은 자동으로 BUYER로 등록되고, 요청받은 판매자는 자동으로 SELLER로 등록된다.
    - SELLING 상태인 상품에 대해서만 채팅방을 생성할 수 있다.
    - 같은 구매자가 같은 상품에 대해 채팅방을 중복으로 생성할 수 없다. (같은 판매자의 다른 상품은 별도 채팅방 생성 가능)
- 채팅방 목록 조회
    - 내가 참여한 채팅방만 조회할 수 있다.
    - 채팅방 목록은 전체 / 구매 / 판매 탭으로 구분하여 조회할 수 있다.
        - 구매 탭 : 내가 BUYER인 채팅방 (내가 채팅을 시작한 채팅방)
        - 판매 탭 : 내가 SELLER인 채팅방 (상대방이 채팅을 시작한 채팅방)
- 메시지 송수신
    - 채팅 메시지는 WebSocket(STOMP) + Redis Pub/Sub을 통해 실시간으로 전송된다.
    - 메시지 전송 시마다 해당 상품의 판매 상태를 확인한다.
        - SELLING 상태가 아니면 메시지 저장 없이 상태에 맞는 시스템 메시지를 채팅방에 전달한다.
            - SOLD : "이 상품은 판매 완료된 상품입니다."
            - CLAIMED : "이 상품은 거래 완료된 상품입니다."
            - CANCELLED : "이 상품은 취소된 상품입니다."
            - EXPIRED : "이 상품은 판매 기간이 만료된 상품입니다."
- 메시지 조회
    - 메시지는 커서 기반으로 조회한다.
        - 최초 입장 또는 재연결 시 최신 메시지 50개를 조회한다.
        - 이전 메시지 더 보기 시 마지막으로 받은 메시지 ID 이전 50개를 조회한다.
- 입장 / 퇴장
    - 채팅방 입장 및 퇴장 시 시스템 메시지("OOO님이 입장/퇴장했습니다.")를 실시간으로 전달한다.
    - 시스템 메시지는 DB에 저장하지 않는다.
    - SELLING 상태가 아닌 채팅방에 입장하면 입력창을 비활성화하고, 입장/퇴장 시스템 메시지를 전송하지 않는다.
    - 퇴장은 다른 채팅방 선택, 로그아웃, 창 닫기(새로고침) 시 발생한다.
